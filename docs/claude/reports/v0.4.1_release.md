# Market Movers Pro - v0.4.1 Release Notes

**Release Date:** 2025-10-20
**Version:** 0.4.1
**Codename:** "Professional Polish"

## Overview

Version 0.4.1 represents a major UI/UX overhaul that transforms Market Movers Pro into a professional-grade trading platform. Inspired by industry leaders like Trade Ideas, Benzinga Pro, and ThinkorSwim, this release delivers a complete visual redesign and adds powerful interactive table features.

This release consists of two phases:
- **Phase 1 (v0.4.0)**: Core Visual Overhaul
- **Phase 2 (v0.4.1)**: Interactive Tables

## What's New

### Phase 1: Core Visual Overhaul (v0.4.0)

#### 🎨 Professional Theme System
- **Dark/Light Mode Toggle** - Instantly switch between professional color schemes
- **Theme Persistence** - Your preference is saved across sessions
- **Custom CSS Properties** - Theme colors defined with CSS variables for easy customization
- **Smooth Transitions** - All theme changes are instant with no flicker

**Technical Details:**
- New `ThemeContext` for React state management (context/ThemeContext.tsx:1-54)
- `ThemeToggle` component with sun/moon icons (components/ThemeToggle.tsx:1-32)
- CSS custom properties in globals.css (app/globals.css:7-131)

#### 📊 High-Density Tables
- **28px Row Height** - See 30-40 stocks at once (previously 15-20)
- **Sticky Headers** - Column headers stay visible during scroll
- **Zebra Striping** - Subtle row alternation for easier scanning
- **Tabular Fonts** - Monospace numbers for perfect column alignment
- **Compact Typography** - 13px body, 11px headers for maximum data density

**Impact:** ~2.5x more data visible on standard monitor

#### 🌈 3-Level Color Intensity
Instantly spot significant moves with intensity-based colors:
- **Dark** (>10% moves) - Bold, high weight
- **Base** (5-10% moves) - Medium weight
- **Light** (2-5% moves) - Standard weight

**Colors:**
- Gains: #059669 (dark), #10b981 (base), #34d399 (light)
- Losses: #dc2626 (dark), #ef4444 (base), #f87171 (light)

**Technical Details:**
- `getColorByIntensity()` function (lib/colorCoding.ts:26-53)
- Applied to all change % and price change columns

### Phase 2: Interactive Tables (v0.4.1)

#### 🔀 Click-to-Sort Columns
- **Three-State Sorting** - Click once for ascending, twice for descending, third time to clear
- **Visual Indicators** - ↑ and ↓ arrows show active sort
- **Smart Type Detection** - Automatically handles numbers vs. strings
- **Works on All Columns** - Symbol, Price, Change %, Volume, etc.

**Technical Details:**
- `useTableSort` custom hook (hooks/useTableSort.ts:1-82)
- Performance optimized with `useMemo`

#### ↔️ Resizable Columns
- **Drag Borders** - Hover over column edge and drag to resize
- **Visual Feedback** - Cursor changes to `col-resize` indicator
- **Minimum Width** - 50px minimum prevents UI breaks
- **Smooth Resizing** - Live updates as you drag

**Technical Details:**
- `useColumnResize` custom hook (hooks/useColumnResize.ts:1-56)
- Mouse event handling with document-level listeners

#### ⏱️ Configurable Refresh Rates
Choose the perfect update frequency for your trading style:
- **5 seconds** - High-frequency traders (warning: high API usage)
- **10 seconds** - Active monitoring
- **30 seconds** - Default, balanced approach
- **1 minute** - Casual monitoring
- **Manual** - Freeze auto-refresh, update on demand

**Controls:**
- Dropdown selector in toolbar
- Freeze/Resume button (visual state: yellow when frozen)
- Manual refresh button (🔄 icon)

**Technical Details:**
- React Query integration with configurable polling (components/windows/ScannerWindow.tsx:20-32)
- `refetchInterval` dynamically controlled

#### ✨ Flash Animations
Real-time visual feedback on data changes:
- **Green Flash** - Price increased
- **Red Flash** - Price decreased
- **Blue Flash** - Volume changed
- **600ms Duration** - Professional timing that doesn't distract

**Technical Details:**
- `useDataFlash` custom hook (hooks/useDataFlash.ts:1-70)
- CSS keyframe animations (app/globals.css:134-168)
- Change detection with `useRef` for previous values

## Known Issues

### Minor Issues

1. **Column Widths Not Persisted**
   - **Impact:** Low
   - **Description:** Custom column widths reset when window is closed
   - **Workaround:** Resize columns after opening scanner
   - **Fix Planned:** v0.5.0 - Save to workspace config

2. **Sort State Resets on Refresh**
   - **Impact:** Low
   - **Description:** Active sort clears when scanner data refreshes
   - **Workaround:** Re-click column header to re-sort
   - **Fix Planned:** v0.5.0 - Maintain sort state

3. **Flash on Initial Data Load**
   - **Impact:** Minimal
   - **Description:** All cells flash when first data loads
   - **Workaround:** None needed (cosmetic only)
   - **Fix Planned:** v0.5.0 - Skip flash on initial render

### Resolved Issues

✅ **CSS Import Error**
- **Issue:** `@import '../styles/themes.css'` caused "Invalid or unexpected token"
- **Fix:** Moved all theme CSS directly into `app/globals.css`
- **Status:** Resolved in v0.4.1

✅ **Server Port Inconsistency**
- **Issue:** Dev server starting on random ports (3001, 3002, etc.)
- **Fix:** Kill process on port 3000 before starting
- **Status:** Resolved - documented in workflow

## Migration Guide

### Upgrading from v0.3.0 to v0.4.1

#### Step 1: Pull Latest Code
```bash
git pull origin main
```

#### Step 2: Install Dependencies
No new dependencies required - all features use existing packages.

```bash
npm install  # Optional, to ensure lockfile sync
```

#### Step 3: Update Environment
No environment variable changes required.

#### Step 4: Clear Browser Cache
For best results, clear browser cache to load new CSS:
```
Chrome: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
```

#### Step 5: Restart Dev Server
```bash
npm run dev
```
